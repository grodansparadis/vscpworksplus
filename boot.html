VSCP Bootloader Wizard

Click on steps or 'Prev' and 'Next' buttons

Info

VSCP Bootloader wizard will now walk you through the steps needed to update the firmware of your remote device

The node to update can be located on a VSCP bus connected to a server on a remote node or be connected to an interface on your local computer.

Please don't turn off or power off the remote node until the firmware update process is completed.

Remote Node

Select device to load firmware to. Enter the nickname or the full GUID for the device you want to work with.

A node that is newly loaded with a boot loader is usually at nickname 0xfe, in most other cases a node stay at the same id (use scan if unsure of which id your node is at).

Algorithm

Select the boot loader algorithm to use.

If you load boot loader info from the MDF the algorithm will be automatically set for you.

Firmware

Select the firmware file to load the remote node with.

You can fetch firmware files which are specified by the MDF or you can load a firmware file from local storage. When loading a local file you must make certain that the file is intended for the remote node. If it isn't and you load it to the remote node it may stop working and if so need manual at location service.
